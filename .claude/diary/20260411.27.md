# 2026-04-11 — Pre-commit speedup, auth fixes, Bun 1.3.12 regressions (Sprint 27)

## What was done

7 PRs merged, v1.1.4 released.

**Pre-commit & build infrastructure:**
- Pre-commit fast path — daemon tests + timing profiler skipped locally, CI-only (#1125/PR #1128). Pre-commit: ~4 min → ~15s. Root cause was daemon tests taking 136s + segfaulting.
- Build codesign workaround — Bun 1.3.12 produces truncated code signatures on macOS `--compile` binaries (oven-sh/bun#29117). `BUN_NO_CODESIGN_MACHO_BINARY=1` + ad-hoc `codesign --sign -` (#1129/PR #1130).
- Bun 1.3.11→1.3.12, MCP SDK 1.27.1→1.29.0, @types/react bump.

**Auth/OAuth fixes:**
- OAuth redirectUrl — `undefined` caused SDK 1.27.1 to skip refresh_token, breaking Atlassian after token expiry (#1122/PR #1127).
- SSE protected resource mismatch — Asana's OAuth metadata advertises `/v2` as resource while SSE endpoint is at `/sse`. Now accepts same-origin URLs (#1120/PR #1131).

**Command fixes:**
- approve/deny wiring — was in `cmdClaudeInternal` (test path only), not in `agent.ts` runtime dispatch (#1121/PR #1132).
- Headless worktree branch prefix — `claude/` namespace prevents collision with existing branches (#1115/PR #1134).
- Orphaned worktree cleanup — IPC failure after `git worktree add` now cleans up instead of leaving ghost worktrees (#1116/PR #1135).

**Also closed:**
- #957, #577, #929 — consolidated into #1004 as the single segfault tracker
- #1124, #1111 — duplicates/already-fixed by #1128

## What worked well

- **Measuring before assuming** — user pushed back on my initial assumption that `bun install` was the bottleneck. Timing each step revealed daemon tests (136s) were 95% of the problem. The fix was trivially obvious once measured.
- **Reusing validated work** — user caught me about to throw away the build.ts stash and have another Claude redo it from scratch. "What would a good manager do?" Pushed the tested code directly instead.
- **QA sessions correctly identified CI segfault as pre-existing** — all 3 QA sessions that hit exit 132 logged it to #1004 and didn't block on it.
- **Fast pre-commit paid off immediately** — sprint commits went through in seconds, not minutes. The fix benefited the sprint that shipped it.

## What didn't work

- **Bun 1.3.12 was a net negative** — upgraded hoping for segfault fix, got a code signing regression instead. Segfaults still happen on CI (Linux). Local macOS passed once but that's non-deterministic. The upgrade added work (#1129) without solving the original problem.
- **Network flakiness killed sessions** — session `6abae14d` (#1121) and QA session `e2577ee8` (#1122 QA) both hit DNS resolution failures to github.com mid-session. Transient — resolved on retry. No root cause.
- **Redundant session spawn** — spawned an opus session to implement #1129 while the identical fix was sitting in a git stash. Wasted $0.21 + a slot. User correctly identified the poor executive function.
- **Stale daemon after build** — daemon was 23 minutes old on a stale build epoch after `bun run build`. Had to manually restart. The pre-flight should compare build epochs, not just check if daemon is running.

## Patterns established

- **Always `open` bun.report URLs** — submits crash telemetry to Bun's team. Added to memory, will add to CLAUDE.md next retro.
- **Don't re-implement validated work** — if you tested a fix locally and it works, push it directly. Don't throw it away and have another session redo it.
- **Measure, don't assume** — timing revealed the actual bottleneck (daemon tests) was different from the assumed one (bun install, coverage overhead).
- **Segfault tracker consolidation** — #1004 is the single tracker. #577 (1.3.10), #929 (1.3.9), #957 (1.3.11) all closed as duplicates.
