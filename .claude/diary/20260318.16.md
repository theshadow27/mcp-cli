## What worked well

- **Clean sweep (15/15)**: First sprint with zero drops or deferrals. Good issue selection — mix of quick wins and meaty features kept slots full without bottlenecks.
- **Worktree reuse via `--cwd`**: When worktrees survived `bye` (uncommitted changes like stale `package.json`), QA sessions reused them via `--cwd`. When auto-cleaned, `--worktree` fallback worked cleanly. No more main-folder pollution (Sprint 15 bug).
- **Pre-commit hook tiers working**: Scripts, hooks, and docs committed instantly. The tiered classification from Sprint 15 (#789) is paying off — no more 95s waits for non-source changes.
- **Adversarial reviews caught real bugs again**: #812 (5 findings — committed cache, non-atomic writes), #697 (nested JSON regex, process.exit race), #785 (3 blockers). Pattern holds: high-scrutiny issues need review.
- **Fast batch throughput**: Batch 1 (5 issues) → Batch 2 (5 issues) → Batch 3 (5 issues) with continuous backfill. No idle slots.

## What didn't work

- **QA #786 took 91 turns / $2.18**: The Zod validation QA session got bogged down in CI failures and multiple merge attempts. Should have been a quick low-scrutiny QA.
- **4 duplicate coverage issues filed**: Sessions implementing serve DX (#765, #766, #811) each filed their own `serve.ts coverage dropped` issue (#824, #825, #826, #831). Need dedup logic or a "check existing issues first" instruction in the implement skill.
- **#718 (WS port reclaim) high token count**: 4843 tokens during implementation — spent a lot of time reading `claude-server.ts` and `ws-server.ts`. The issue was well-specified but the code was complex. Not a waste, just expensive exploration.

## Patterns established

- **Sprint 15 process fixes validated**: `--worktree` fallback for QA, timing budget as warn-only, pre-commit tiers — all worked as intended. No regressions.
- **Hash-based timing cache shipped**: `test-timings.json` is gitignored, incremental profiling only re-times changed files, `--full` flag for complete re-timing. The Sprint 15 $2700 cost explosion cannot recur.
- **Duplicate issue detection needed**: 4 dups in one sprint is too many. Add "search for existing issues before filing" to the implement skill.
