# Sprint 23 Retro — 2026-04-08

## What happened

Sprint 23 shipped the scoping epic (#981, all 5 sub-issues), quota monitoring (#1008), daemon flock fix (#1053), and 8 other fixes. 15/15 merged. v1.1.0 released.

This was the smoothest sprint in recent memory — no laptop sleep incidents, no runaway sessions, no quota exhaustion. The flock fix (#1053) means duplicate daemons after sleep/wake should be gone for good.

## The scoping epic (#981)

All 5 sub-issues landed in dependency order across batches 1-3:
1. #1009 (scope registry) → batch 1
2. #1010 (scope detection) → batch 2 (after #1009 merged)
3. #1011 (session filtering) + #1012 (config parent matching) → batch 2 (parallel, after #1010)
4. #1013 (mcpctl scope selector) → batch 3 (after #1011)

The dependency chain worked perfectly — each issue unblocked the next without blocking slots. The ~300 LOC estimate from the design was accurate.

## Quota monitoring (#1008)

The OAuth usage endpoint spike from Sprint 22 paid off — the implementation was straightforward because the API was already proven. The key was the `anthropic-beta: oauth-2025-04-20` header discovered via mitmproxy during the spike.

## The flock fix (#1053)

Adversarial review caught a real bug: `openSync("w")` truncates the PID file BEFORE acquiring the flock, zeroing the running daemon's PID while it's live. The repair fixed it with `O_WRONLY|O_CREAT` (no truncate), then truncate after lock acquisition. Good catch.

## What worked

- **Dependency chain in batches** — scoping epic's serial dependencies flowed naturally through the batch system without wasting slots.
- **Adversarial review sticky comments** — #1053 review caught the O_TRUNC-before-flock bug. #1023 review caught that the test was using mocks that bypassed the actual fix code. #1024 review caught the missing process timeout.
- **Pre-commit fixes first** — putting #1014/#1026/#1015 in batch 1 unblocked all subsequent sessions from pre-commit failures.
- **Admin merge for Bun segfaults** — stopped wasting time retrying CI. The upstream fix (oven-sh/bun#27960) is still pending.

## What didn't work

- **`wait` idle detection appeared broken during sprint** — polling with `sleep + ls` throughout because `wait` wasn't detecting idle transitions. Post-sprint investigation: the fix IS working on the v1.1.0 build. The issue was running against the old daemon (protocol mismatch) for most of the sprint before the final rebuild. Lesson: rebuild + restart daemon as the FIRST step of a sprint, not just pre-flight version check.
- **Bun segfaults escalated to 33%** — 12 of 36 CI runs segfaulted (up from 2-3% in Sprint 22). 12 of 14 failures were Bun segfaults, only 2 were real. This is now a P1 — every third CI run fails. Possible cause: more worker-spawning tests added (mock provider, scope tests). The upstream fix (oven-sh/bun#27960) is still open. Updated #1004 with the escalation data.
