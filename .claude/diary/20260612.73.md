## What worked well

- **The stdio arc sequencing was load-bearing**: stderr capture (#2738) merged
  hours before the #2688 investigation needed it, and the investigator's first
  move was reading the child's actual stderr. Plan-time dependency edges paid
  off exactly as designed.
- **Verify-the-hypothesis discipline (new memory, applied twice same-day)**:
  #2740's implementer disproved the investigation's teardown-race finding
  (budget-watcher never closes the db; the 2 closed-db errors are an
  intentional fallback test) and fixed the real wall-clock race instead — then
  documented the disproof on the go/no-go record. #2729's worker reproduced
  the flake (run 6/20) before fixing, and found the actual mechanism differed
  from the issue's hypothesis (poll-timer interference, not touch/prune skew).
- **Reviewer self-repair pattern**: 4 PRs (#2769, #2779, #2790, plus #2696's
  implementer redesign) repaired by the session holding the diagnosis, each
  verified by a fresh QA — zero self-approvals, fast turnaround.
- **The gate lease (#2690) merged first and the rest of the sprint ran under
  it** — 12+ concurrent sessions, zero SIGTERM storms, zero CPU wedges.
- **`mcx daemon reload` dogfooded immediately**: split-brain detector → its own
  recommended command → clean reload, three times.
- **Event-driven orchestration**: ~80 monitor notifications drove every
  transition; zero blind polling; phase-machine state stayed consistent
  (one force-write, documented below).

## What didn't work

- **#2762's fix shipped a worse regression than the bug it fixed** (#2721 →
  #2796): its QA validated list membership via unit tests + CI, but nothing
  boots the compiled binary, so a count-dependent Bun outbase quirk took out
  all five workers silently. Concrete fix shipped: post-compile boot smoke in
  the build itself + `daemon-workers.spec.ts` pinning `--root`; #2800/#2801
  track guard hardening and the durable manifest-resolution exit. Lesson: for
  build-mechanism changes, the verification must exercise the ARTIFACT, not
  the source.
- **Plan amendment bypassed hot-file analysis** (#2768): #2754 was added at
  16:10 after the overlap matrix was computed; it and #2719 both edit
  `ci-steps.ts` and were launched in parallel — caught at PR time, serialized
  manually (zero-conflict rebase, composition QA'd). Fix direction filed:
  re-run overlap check on every amendment.
- **Phase-graph drift**: `repair → qa` rejected by the hardcoded
  `VALID_TRANSITIONS` fallback while `.mcx.yaml` allows it — my
  `work_items_update` omitted `repoRoot` and hit the fallback (#2781). One
  documented force-write to resync. Also: the CI-red escalation flow
  (repair → fresh review → QA) has no graph representation.
- **QA prompts with orchestrator-guessed branch names**: I twice typed a
  branch name into a spawn prompt instead of reading it from the phase
  output/PR (caught both, one needed a correction send). The phase scripts
  emit the right prompt — use it verbatim.
- **#2769 merged carrying a stale `review:changes` label** (by-design after
  self-repair + QA verification, but it reads as a contradiction in history).
  Cleaned post-merge on #2769/#2779/#2790. The label protocol for
  self-repaired reviews could use a tightening pass.

## Patterns established

- **Reviewer self-repair + fresh-QA-verifies** is now the default route for
  contained review findings (1–3 edits with file:line diagnoses); fresh opus
  repair reserved for redesigns (#2696) and qa:fail blockers (#2754).
- **Empirical disproof is a valid gate outcome**: an investigation-prescribed
  fix can be satisfied by documented refutation (#2740 → go/no-go item (c)),
  provided the disproof is itself adversarially verified.
- **Build guards must boot the artifact**: `smokeDaemonWorkers()` runs inside
  `bun run build` — the class of "compiles green, dead at runtime" worker
  regressions (#2721, #2796) now fails the build.
