## What worked well

- **Adversarial review caught real bugs on every high-scrutiny PR.** #2580 sanitizer: an `aws-secret-key` regex matching every 40-hex git SHA (false-positive bomb), a pre-commit hook silently skipping binary files (false security guarantee), `SecretMatch.matched` leaking raw secrets in the exported type. #2583: sha256 computed-but-never-compared (the "verified" install verified nothing). #2584: a `highestKnown` reduce that broke patch tracking. #2588: a session leak on timeout. QA also caught substantive bugs on nominally-low-scrutiny PRs — #2582's always-true verification tests, #2565's hidden production stdio-restore bug (→ #2602/#2234).
- **The convergence→simplify rule fired correctly on #2580.** review(3 blockers) → QA(4) → Copilot(5), all *new* coverage gaps = reactive regex-listing that wouldn't converge. Stopped patching, ordered a redesign to a principled taxonomy; the redesign then survived an adversarial probe (no missed class) — convergence achieved.
- **Reviewer self-repair** handled most contained findings without spawning fresh repairs, keeping the worktree+diagnosis loaded.
- **Branch-handoff via `--cwd` + `--keep-worktree`** held across impl→review→repair→QA without losing branches, including across the overnight pause.

## What didn't work

- **The bun allocator wedge (#2597) dominated the sprint.** Parallel sessions running `bun run am-i-done` drove load to **117** on a ~10-core box. Nerd-snipe (general-purpose agent + /nerd-snipe-skill, per user request) found the root cause: a zero-backoff `madvise(MADV_DONTDUMP)` EAGAIN busy-spin in bun's bmalloc `SYSCALL` macro — under memory pressure a `bun test --test-worker` child pegs a core forever, ignores `--timeout` (spin is post-test-body) and SIGTERM (SIGKILL-only). Upstream bun#27490/#27766 unfixed/unreleased. Band-aid: an in-session elapsed-time (>90s) reaper. **It then escalated** from local CPU saturation to **hanging the required `coverage` CI job** (no `timeout-minutes` → 6h squat), which is what blocked #2587/#2588 and forced the roll. Fix is sprint 70 P1 #1: concurrency cap in am-i-done + SIGKILL-pgroup watchdog + productize the reaper.
- **PROCESS BUG: raising an AskUserQuestion on quota suspended the sprint overnight.** At 100% 5h-quota I asked the user "wind down vs pause vs finish-2" — but sprints run unattended, so the context sat idle for hours doing nothing. Quota exhaustion is predictable (`resetsAt` is in the payload); it must be a self-executing policy. Fix: promoted into run.md's Quota-gating section this retro (finish no-LLM actions → ScheduleWakeup to resetsAt → auto-resume; never AskUserQuestion for quota). Memory `feedback_quota_autonomous_policy`.
- **CI fragility under the wedge.** Test jobs (`check`/`coverage`/`check-no-claude`) have no `timeout-minutes` → a wedged job squats to GitHub's 6h default. I also wasted effort on a wrong fail-fast theory (ci.yml jobs are independent; my own cancel+rerun churn created the "cancelled coverage" state). Both → sprint 70 CI hardening.
- **Reaper collateral**: the >90s elapsed reaper occasionally SIGKILLed legit slow coverage workers, sending #2588 into a long self-misdiagnosis spiral (12k turns, $19.5) before I told it the hang was infra, not its code. A productized reaper must gate on load or provable-wedge, not raw elapsed time.
- **#2587 scope ambiguity**: "replay through the mock" — impl did state-machine validation, QA wanted a spawned mock subprocess. Resolved by reading the issue's outcome-based Done-means (met) and keeping the stronger interpretation as enhancement #2616, rather than looping. Worth tighter "Done means" wording up front.

## Patterns established

- **Quota = autonomous policy, never a human gate** (now in run.md).
- **Non-required-but-flaky CI checks must not block merges**: verified via `mergeStateStatus: UNSTABLE` + the ruleset's required set (check/coverage/build) that check-no-claude is non-required, and merged past it.
- **Coverage-type convergence**: a security/coverage component surfacing a new *missing case* each round (vs the classic type-laundering treadmill) is also a legitimate simplify trigger — anchor on an established taxonomy from the start.
