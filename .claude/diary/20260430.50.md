## Patterns established

- **Long-lived Monitor is the runbook default, not an alternative.** `references/run.md` should invert the framing: lead with the persistent stream, present `--max-events 1` as the subprocess-harness fallback. Filed as runbook update for next sprint (mentioned in retro notes; codification deferred).

- **Reviewer self-repair scales with finding count + scope.** 1–3 contained file:line edits → reviewer fixes its own. 4+ findings or any architectural call (remove this whole mechanism, unify this API surface) → impl session. The 8-finding PR #1950 confirmed the boundary.

- **Worktree cleanup gracefully handles staged residue from merged-elsewhere branches.** Pattern: `git stash -u` then `git worktree remove`. Adopted after Hank's 19-modified worktree post-#1856 merge.

- **Author-trust filter at survey time, not spawn time.** Sprint 50 codified `--author theshadow27` in plan skill Step 1.
