## What worked well

1. **Citation-counting beats embeddings for "what to answer first"** — the verification swarm independently encoded a dependency graph via cross-refs. Counting inbound REQ citations found the #1 leverage point (`REQ-05-001-01`, fan-out=19) without any semantic analysis. Embedding clustering on questions themselves surfaced only 27 true duplicates in 1,219 questions
2. **Three-tier auth debugging** — walking `clientInformation()` tiers (opts → SQLite → Keychain → DCR) explicitly found the real cache. Clearing just SQLite was insufficient because Claude Code's keychain tier was the actual poisoner
3. **Keychain surgery pattern** — backup → read → JSON edit single key → `security add-generic-password -U` → verify. Touching one key inside `mcpOAuth` without disturbing `claudeAiOauth` is surgical and reversible via the backup
4. **Filing targeted issues mid-sprint** instead of patching in-place — told orchestrator #1546 is P0 so the sprint flow picks it up normally

2. **CI `pull_request` trigger didn't fire** for #1513/#1515, gating on #1506. Required manual `gh workflow run` to unblock. Root cause not yet filed

4. **The fix for the fix collided** — #1076 (scope fallback) and #1122 (no-port redirect fallback) individually correct, together poison the cache path that `server-pool.ts:299` exercises. Two independent "fixes" created a third bug. Filed #1517 to capture the interaction
5. **DCR clients cached before success** — both Claude Code and mcx persist `client_id` the moment DCR returns, before the flow actually completes. First consent bounce or authorize 500 → zombie cached forever. Same class of bug in both codebases (#1517 for mcx, #1546 guards the read side for both)
6. **`expiresAt=0` read as valid** — `readKeychainTokens` guard only checked whether the entry was expired, not whether it had any usable token at all. Falsy `expiresAt` + empty tokens = "valid" entry returned

## Patterns established

- **Cache only after success**: don't persist OAuth `client_id` / DCR artifacts until `saveTokens` succeeds. Pre-authorization state must not survive the session. Applies to both SQLite (mcx) and Keychain (Claude Code) tiers
- **Usability guard on cached auth reads**: `keychain.ts` (and parallel SQLite readers) must verify `entry.refreshToken || (entry.accessToken && entry.expiresAt > now)` before returning. Bare `expiresAt` checks accept zombies
- **Citation-counting for triage**: when a swarm files many cross-referencing issues, inbound-citation count on the primary ID (REQ, module, etc.) identifies the highest-leverage unknowns without any LLM spend. Pure graph structure
- **Decisions live with the work**: when answering investigate-style questions, write the decision as a `## Decision` section directly in the issue file + `status: decided` in frontmatter, with `closed_by: <parent-id>` / `downgraded_by:` on cascaded issues. Don't build a parallel decision tracker — git log on the issue directory is already the changelog
- **Keychain surgery over wholesale delete**: for shared credential blobs (`Claude Code-credentials` holding both `claudeAiOauth` and `mcpOAuth`), dump → edit single key → `security add-generic-password -U` → verify `claudeAiOauth` byte-equal. Never delete the whole item
- **Private memory symlinks per project**: `~/.claude/projects/<slug>/memory → ~/<repo>/.claude/memory`. Keeps session-local runbooks addressable from the project dir and the session dir interchangeably
