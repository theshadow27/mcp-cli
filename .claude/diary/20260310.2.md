## What worked well
- **Adversarial reviews caught real bugs**: Shell injection in worktree hooks, stdin flush missing in codex, timer leaks, cross-process stderr contamination. Every high-scrutiny PR had blocking issues found.
- **`--short` flag was high-value**: Immediately useful for monitoring. The verbose wait output was burning ~10k tokens per sprint on status polling alone.
- **Closing stale epics**: #170, #181, #183 were long-complete but open. Housekeeping reduces cognitive load on board surveys.
- **Customer issue turnaround**: #526 filed → implemented → reviewed → repaired → merged → bug found (#546) → fixed → merged, all in one sprint.

## What didn't work
- **ExitPlanMode not in `--allow`**: #526 got stuck in a permission loop. `/implement` enters plan mode, needs ExitPlanMode permission. Now documented in MEMORY.md.
- **Triage script was silently broken**: `--pr` flag never worked — returned zeros for everything. All PRs in this sprint got routed to low scrutiny. Fixed mid-sprint (#533).
- **#546 repair cost $401**: Session ran 1249 turns. Likely looped on the bug fix. Need cost guardrails or turn limits.
- **Worktree cleanup lost #539 work**: First #539 session completed but `bye` cleaned up the worktree before PR was pushed. Had to respawn. Added "don't bye before verifying PR pushed" to MEMORY.md.

## Patterns established
- **Always include `ExitPlanMode` in `--allow`** for `/implement` tasks.
- **Manual triage for this sprint**: `gh pr view --json additions,deletions,changedFiles` + eyeball. High if: additions+deletions >= 120, additions >= 100, 4+ src files across 2+ packages, or touches risk areas.
- **`--short` for monitoring**: Use `dist/mcx claude wait --short --timeout 60000` in all monitoring loops going forward.
- **Don't `bye` before verifying PR pushed**: Worktree cleanup destroys unpushed work.
- **Codex arc is serial**: #303 (spike, done) → #305 (provider, done) → #531 (daemon, done) → #306 (CLI, next).
