## What defined this quarter

### The adversarial review system matured

Sprint 20 introduced sticky PR comments and delta mode for re-reviews. By Sprint 24, the system was catching real regressions — the OAuth scope fix went through 2 review+repair rounds, each catching bugs that would have broken production (unconditional scope on token exchange, fallback defeating SDK cascade).

**Pattern validated:** Adversarial review on sonnet, repair on opus, sticky comments as shared context. Delta mode prevents reviewers from repeating fixed issues.

### The nerdsnipe skill proved its value

- **#978 (idle detection):** Found that strict Zod schemas silently dropped NDJSON messages, preventing state transitions. Then audited ALL parse sites and found 3 more with the same bug class.
- **#1008 (quota API):** Three attempts — first was garbage, second found the endpoint but wrong auth, third (with user-intercepted traffic) found the `anthropic-beta` header. Final result: working quota monitoring.
- **#1109 (worktree pollution):** Found `cwd: undefined` root cause and `core.bare` flip.

**Pattern validated:** Detailed investigation instructions produce better results. "NOT acceptable" sections prevent lazy conclusions.

### Infrastructure issues were the real enemy

Not features, not bugs — infrastructure:
- **Bun segfaults** — 33% CI failure rate by Sprint 23. Still upstream (oven-sh/bun#27960).
- **Pre-commit timeouts** — every session in Sprint 24 needed `--no-verify`. Fixed in Sprint 25.
- **Worktree pollution** — files leaking to main repo across sprints 21-26. Root cause found in Sprint 26.
- **Stale daemon** — version mismatches after rebuild causing silent failures.
- **Laptop sleep** — kills sessions, corrupts state, flips `core.bare`.

### The orchestration model works but has limits

**What works:** Parallel spawning, adversarial review, event-driven wait, quota monitoring, session scoping. The sprint skill can run 15 issues through implement→review→repair→QA autonomously.

**What doesn't scale:** Polling for idle detection (the #978 fix helped but isn't perfect). Manual `--no-verify` approvals. Worktree cleanup. Session cost tracking (garbage numbers). The orchestrator's context window is the bottleneck — at 90+ PRs, earlier sprint context is compressed.

## Lessons learned

### Process
1. **Rebuild + restart daemon as first step of every sprint.** Not just pre-flight — first step.
2. **Don't bye investigation sessions** until the user has reviewed findings.
3. **File every problem immediately.** The "file everything" rule caught 40+ follow-up issues across 7 sprints.
4. **Scrutinize spike findings.** The first #1008 spike was wrong because it wasn't cross-referenced with known issues (broken cost tracking, Max plans).
5. **Pre-bye tree checks** catch worktree pollution before it compounds.

### Technical
1. **Strict Zod schemas at protocol boundaries are time bombs.** Use safeParse + fallback + logging.
2. **git worktree operations can flip `core.bare`.** Guard after every add/remove.
3. **OAuth scope resolution has a 6-tier cascade.** Don't shortcut it — the shortcuts break other servers.
4. **`flock()` on PID file is the correct daemon singleton pattern.** PID existence checks have race windows.
5. **The mock agent provider** is the foundation for reliable CLI testing without API costs.

### Collaboration
1. **Session survey data drives prioritization.** "220+ attempts to use mcx <server-name>" is more compelling than "this would be nice."
2. **The user sees things the orchestrator misses.** `core.bare` was caught because the user asked "is it getting worse?" — the orchestrator was just resetting files.
3. **Context is gold.** The quota API investigation only succeeded because the user intercepted actual traffic with mitmproxy and found the missing `anthropic-beta` header.
