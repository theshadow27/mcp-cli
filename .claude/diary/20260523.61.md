## What worked well

- **Triage routing was accurate.** Every low-scrutiny PR went straight to QA and passed (most first-try); the one high-scrutiny goal (#1312) went to adversarial review, which immediately flagged the false-atomicity no-op `rollbackApplied()`. No misroutes.
- **Holding auto-merge for Copilot re-reviews caught real bugs — twice.** #1312 round-2 Copilot re-review surfaced 4 genuine bugs (frontmatter not stripped on create, create→modify duplicate-create coalescing, per-ref error dedup, atomicity wording) — none stale re-flags. #1281's Copilot found a real cross-ref `treeCleared` corruption (a global bool where `deleteall` is per-ref). Both would have merged broken under blind auto-merge.
- **Investigation gate paid off on #2255.** Rather than bumping the 4s timeout, the worker root-caused an actual `killPid` skip bug under load. The "find the mechanism, don't band-aid the symptom" gate produced a real daemon fix.
- **Dependency serialization via blockedBy + rebase-on-merge.** The writer chain (#1277→#1263→#1262) and parser pair (#1280→#1279) shared files; blockedBy edges + "rebase on merged sibling" nudges kept them conflict-free.
- **Orchestrator-side redundancy check.** #1323 was closed directly (with a diff citation) once #1311 merged the identical fix — no wasted session.

## What didn't work

- **Flaky `#940` daemon suite was the dominant friction.** Its 4s deadline (and the real kill-skip bug) failed CI on #2243, #2254, #2257, #2241 — bimodal (fast pass or 4s-ceiling fail), ~50% under load. Each cleared via `gh run rerun` or a rebase-for-fresh-run, costing many orchestrator cycles before #2255/#2268 fixed it at the root. **Fix applied:** root-caused and merged the daemon fix same-sprint; future sprints won't hit it.
- **Byeing a session before its successor checked out the branch deleted the local branch.** Hit 3× on #1281 (impl→QA, QA→repair, repair→QA-r2): plain `mcx claude bye` removes the worktree and its local branch, so the next phase's fresh `--worktree` session couldn't `git checkout` it and had to re-fetch from the remote with a nudge — wasted turns. This was self-inflicted: I spawned each phase with a fresh `--worktree` and byed without `--keep-worktree`, ignoring the existing run.md invariant to reuse worktrees via `--cwd`. **Fix applied:** rewrote the run.md branch-handoff rule — `bye --keep-worktree` + spawn the successor with `--cwd <worktree>` so one worktree carries the branch through impl→review→repair→QA; only a no-flag `bye` (which deletes the worktree) after the PR merges. (`mcx claude bye --keep-worktree` already existed — I just wasn't using it.)
- **#1312 needed two repair rounds.** Round 1 fixed the 6 adversarial blockers but its claimed fixes for multi-commit-same-path / ignored-newVersion were incomplete or re-flagged by a fresh Copilot pass; round 2 closed them. High-scrutiny transactional code is genuinely hard — the two-round cost was warranted, not avoidable churn.

## Patterns established

- **Re-review gate for high-scrutiny correctness PRs:** after repair, do NOT arm auto-merge until a fresh Copilot/QA pass confirms the fixes; disarm auto-merge the moment new substantive inline threads land. (Caught real bugs on both #1312 and #1281.)
- **Branch handoff between phases:** treat the PR branch as living on the remote — successor sessions fetch+checkout rather than assuming the local branch survives a `bye`.
