## What worked well

- **Pre-flight `mcx claude wait` smoke-test caught the P1 immediately.** Five Batch-1 sessions died in <1s with `spawn exited`, daemon log showed `Pruned dead session ... pid X no longer alive`, single direct `claude` invocation from the orchestrator surfaced the full rejection error: *"host 'localhost' is not an approved Anthropic endpoint."* From "every spawn dying" to "P1 root-caused" was ~5 minutes.
- **Reverse-engineering recon paid for itself.** Two parallel agents (backup + binary-decompile) returned in ~5 min with: 5-host allowlist verbatim from binary, pure URL.hostname string-match (no DNS — so `/etc/hosts` works), TLS enforced, no env bypass, no integrity check, the patch-call site identifiable by unique string `tengu_sdk_url_host_rejected`. That recon directly informed the user's separate-session response (length-preserving IPv6-loopback substitution rather than naive byte-NOP).
- **Self-repair pattern continued paying.** 4 self-repairs this sprint (#1768 thread replies, #1800 timeout/CLOSED fixes, #1802 thread replies, #1746 thread replies + 2 new tests). Saved ~$15-20 of fresh-opus repair sessions. Reviewer/QA workers know their own findings best.
- **Force-pin via wrapper script + `chflags uchg` finally stuck.** Two prior attempts at re-symlinking to 2.1.119 were stomped by the auto-updater within minutes. The third attempt used a small shell script at `~/.local/bin/claude` with `chflags uchg` set — the auto-updater can't replace it.
- **Long-lived sprint-{N} branch worked as designed.** Single PR (#1805) accumulated plan, start timestamp, results, release commit, and the upcoming retro diary. One watchable thing for the user, no `release/vX.Y.Z` short-lived branch churn.

## What didn't work

- **The sprint plan badly underestimated #1798.** It was tagged "low scrutiny / sonnet / text-only fix" — turned out to be a 9-file sweep (`worktree-shim`, `agent`, `claude`, `gc`, `output`, plus 4 spec files) that even after Carol's first pass missed 14 more `printError`-success-message sites that Hank's review caught. Final cost was $4.55 — the most expensive impl this sprint by 2× — for what was supposed to be the cheapest. **Improvement**: triage prompts should ask "are there other sites with the same anti-pattern?" before spawning. File a meta issue.
- **Auto-updater stomping the symlink.** Anthropic's update path is aggressive — the `~/.local/bin/claude` symlink got rewritten to 2.1.121 twice during sprint pre-flight before I escalated to `chflags uchg`. **Fix landed**: archived 2.1.119 (and .114-.118) to `~/.local/share/mcp-cli-archive/claude-code/` outside auto-updater scope, with sha256s in a `README.txt` for restore.
- **The orchestrator's daemon ran the pre-#1797 build the entire sprint.** Restarting to pick up the merged fix would have killed all in-flight sessions. Workaround: every `work_items_update` to a manifest phase needed `force=true` until #1797 merged, after which only NEW transitions (involving phases not yet known to the daemon's cached manifest) needed force. Annoying but never blocking.
- **#1800's implementation overshot the plan.** Plan said "doc + workaround in run.md (or wrapper if implementer wants — file as separate enhancement)". Dave wrote the wrapper directly + tried to update `run.md`, which is a meta file. Two corrections needed: revert the meta hunk (`#1816` filed for next-sprint pickup) and adversarial review on the larger-than-expected diff. Final result is good but cost a round-trip. **Improvement**: the implement prompt could be sharper about meta-file protection — Step 1a from PR #1804 helps but didn't catch this.
- **Phase poller lag → repeated "qa → spawn" output.** After every `qa:pass` label set, the first `mcx phase run qa` tick still saw the stale `reviewStatus: "pending"` and returned a spawn command. The retick within ~10s saw the updated label and returned `goto done`. Lost ~30s per merge. Not a blocker, but a UX papercut.
- **`session:stuck` false positives still everywhere.** The fix landed (#1799/#1815) but the running daemon predates the fix, so we got dozens of stuck events for sessions running long bun-test suites. Each one triggered an `mcx claude wait` early-return — pure noise. Will be quiet next sprint.

## Patterns established

- **Out-of-band sprint adds work.** When the user authored two parallel-session P1 PRs (#1826, #1830), the orchestrator pulled them in as opus-QA picks via `mcx track <issue> + force phase=qa + spawn /qa`. Both merged cleanly. Pattern: any user-authored PR ready for review can be ingested mid-sprint with `force phase=qa, spawn opus QA`. Doesn't need a new sprint.
- **Trivial-rebase resolution stays with the author.** When PR #1830 had a 5-line content conflict in `constants.ts` (both PRs added a different entry to the same object), the orchestrator inspected, reported "trivial — both lines, keep both", and waited for the author to resolve in their separate session rather than ferrying through the QA worker's worktree. Faster + clearer ownership.
- **Don't blanket `mcx gc` or `mcx claude worktrees --prune` when other sessions are active.** The user explicitly noted they were running #1808 in a separate Claude session. Sprint cleanup should never use blanket prune; bye+remove only the sessions the orchestrator spawned.
