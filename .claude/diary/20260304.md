## What worked well

1. **Breaking oversized issues into phases** — #55 was too big for one pass; splitting it into 5 focused issues (#55, #83-#86) let the implementer ship the security fix without getting blocked on UX polish.
2. **Security review flagging the design flaw early** — the review session reframed #55 from "add an approval prompt" to "remove auto-loading entirely," which is the right fix. The architectural discussion happened before implementation, saving a rewrite.
3. **Keychain reuse for OAuth** — discovering that Claude's stored tokens work directly in the CLI (same client ID, same OAuth grant) is a significant DX win. Zero friction for servers already authed in Claude native.
4. **QA session catching the socket gap** — the QA pass on #64 found that `mcpd.sock` had no explicit permission hardening even after `~/.mcp-cli/` was fixed. The audit function only checked dir + db, not the socket.

## What didn't work

1. **Parallel in-flight work on overlapping files causes branch collisions** — sessions 8d6ae066 and 58787b8b both touched `fs.ts`, `loader.ts`, `constants.ts`. A hook stashed changes and switched branches mid-session. Both sessions had to rebase on latest main and redo edits. Need better worktree isolation for concurrent implementations.
2. **Compiled binary cache** — fixing the `127.0.0.1` bug in source had no effect because the daemon was running the compiled `dist/mcpd`, not the dev source. Required `bun run build` + daemon restart. Bit the user twice before they caught it.
3. **Stale OAuth client registration** — the SDK cached the port-0 client registration in SQLite and reused it on retry instead of re-registering. No automatic invalidation. The user had to manually notice the wrong port in the URL to diagnose.
4. **Session 7fb70dd0 ended on ambiguous state** — QA on #56 revealed the socket wasn't hardened, but the fix was left for #56 to implement rather than patching it inline. The socket `chmodSync` is one line; leaving it for a separate issue added a gap.

## Patterns established

- **Match OAuth URL precisely to Keychain entry** — the Keychain reader does exact URL matching. When reusing Claude's OAuth tokens, the server URL in `servers.json` must match what Claude stored (often the SSE endpoint, not the newer v2 URL).
- **Always rebuild before testing daemon auth flows** — OAuth changes in auth code require `bun run build` + daemon restart. Dev source changes are invisible to the running compiled binary.
- **Socket permissions are separate from directory permissions** — `chmod 0700` on the parent dir provides traversal protection but does not set socket permissions. Must `chmodSync(socketPath, 0o600)` explicitly after `Bun.serve({ unix })`.
- **Re-auth resilience requires persisting the client ID** — `mcp add --client-id` stores the ID in `servers.json`; the OAuth provider reads it as the highest-priority source for `clientInformation()`. Without this, token expiry = manual re-setup.
