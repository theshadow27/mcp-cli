## What worked well

1. **Registry discovery from docs page source** — User pasted JS from the Claude docs page which had `https://api.anthropic.com/mcp-registry/v0/servers` hardcoded. Fetching directly gave 100+ servers with slugs, tool names, transport URLs, `claudeCodeCopyText`. Rich enough to build search + install immediately.
2. **#74 landed before #75 was implemented** — `addServerToConfig()` from `config-file.ts` was ready to use as the install write path without coupling the two implementations together.
3. **QA sessions adding targeted tests** — Each QA pass found coverage gaps and added specific tests (error paths, edge cases, short flag aliases) without re-testing things the implementer already covered. Tests went 419 → 457 → 525 across the three QA sessions.
4. **Bun.build plugin approach for WASM** — Patching jq.js at build time to inject `wasmBinary` directly is cleaner than the phoenix-octovalve approach of externalizing + copying to runtime. True single-binary with zero filesystem extraction.

## What didn't work

1. **jq WASM CJS thenable bug was deeply non-obvious** — Three separate broken approaches before finding the root cause: (1) base64-extract-then-import failed because bundled jq.js had baked-in `__dirname`; (2) `import("jq-web")` returned a `__toESM`-wrapped Promise with copied `.then`/`.catch` making it a broken thenable — calling `.then` on it threw "|this| is not a Promise"; (3) patching out the first `module.exports = jq` was right but got there after significant wrong-path debugging. Final fix: also wrap the export as `{ ready: Promise }` so `__toESM` can't turn it into a broken thenable.
2. **Minification hid the CJS double-export bug** — Couldn't inspect the bundle until minification was turned off. Should turn off minification earlier when debugging bundling issues.
3. **Bun `.env` doesn't load from binary directory** — User expected `.env` next to the server script to be picked up. Bun only loads `.env` from CWD. When the compiled daemon binary runs, CWD isn't the install directory.
4. **QA session staging extra files** — Pre-commit hook pulled in modified files from parallel OAuth work (#76) that happened to be staged. Commit included more than intended but was still valid.

## Patterns established

- **Discover registry APIs from public docs pages** — docs pages often embed API calls directly in React components; reading the page source gives the full endpoint, parameters, and response shape.
- **Build plugin for WASM** — when embedding WASM in a Bun compiled binary: use a `Bun.build` plugin to patch the Emscripten loader inline rather than externalizing and copying. Inject `Module['wasmBinary']` before the module runs.
- **Wrap CJS Promise exports as `{ ready: Promise }`** — prevents `__toESM` from turning a CJS Promise export into a broken thenable by copying `.then`/`.catch` methods with wrong `this` binding.
- **Turn off minification first when debugging bundles** — saves multiple debug rounds.
