**Stdio drop-and-hang class (the #2825 lineage, closed structurally):**
- Narrowed the #2814 guard so a `result` frame buffered before an involuntary disconnect is honored instead of dropped (#2838 / #2852) — round-1 adversarial review caught that honoring in `ended` state would resurrect torn-down sessions and emit phantom results through wildcard waiters; repair narrowed to `disconnected`-only
- Bounded the stdioDrainDone await with `Promise.race([drain, workCompletedSignal])` — cancellation tied to the state machine, not a wall-clock cap, so a grandchild holding the stdout fd can't hang the exit handler and completion is proven before early-stop (#2833 / #2878); residual no-result corner filed as #2879

## What worked well

- **The investigation gate paid for itself twice.** #2837's nerd-snipe produced a reproduced mechanism + fix plan for ~$1.73 before any impl spend; the impl then shipped in one round. And #2788's implementer independently disproved the issue's prescribed root cause against the artifact instead of cargo-culting it — the "verify investigation hypotheses" rule executing as designed, both directions.
- **Reviewer self-repair + fresh-verifier flow ran clean 3×** (#2838, #2837, #2840): round-1 reviewer fixes its own contained findings, leaves the label, a fresh round-2 session verifies. All three round-2 verifications confirmed the repairs from the diff, none rubber-stamped.
- **Flake discipline held.** The #2851 coverage failure was NOT rerun-until-lucky: flake-patrol re-derived the mechanism (legacy 1000ms budget vs wall-clock timer under coverage load, hand-set in #589 predating the 1500ms default), correcting the QA's wrong shutdown-latency theory, and the merge waited for 3 clean reruns. Both qa:fail overrides carry mechanism-named comments.
- **Quota pause and resume was clean.** ~7.5h pause at 5h-window exhaustion; on reset, all 18 idle sessions were reconstructed from PR/label/commit state and nudged; nothing was lost or double-spawned.
- **Event-driven orchestration**: the Monitor stream drove every transition; zero polling loops, zero `mcx claude wait` calls.

## What didn't work

- **QA passed a data-loss bug on a destructive command** (#2662 / #2873): the closed-PR reclaim path had no ahead-of-remote guard — a clean-tree worktree whose branch had unpushed commits would be removed and `-D`'d. Caught only because the orchestrator read the diff after qa:pass (the "resolved = MERGED | CLOSED" line smelled). Repair added a fail-closed `isAheadOfForge` guard; round-2 QA verified all three error paths. Lesson applied: destructive-surface PRs got explicit adversarial-probe checklists in the QA brief, but the checklist item ("must not false-positive on closed-without-merge") was answered by the QA at the wrong altitude — it verified dirty-tree protection, not committed-but-unpushed. Follow-up: consider a `review` phase (not just QA) for any PR that deletes user state, regardless of metric scrutiny.
- **Two "Bun crash" mischaracterizations in one sprint.** #2863's attempt-1 coverage failure was called a post-suite Bun crash by QA and by my own override comment — the artifact showed a clean ` 1 fail` from a zero-headroom `pollUntil(...,5000)` at cli-orchestration.spec.ts:554 (filed #2877, override rationale corrected on the merged PR). Characterize from the artifact, not the job summary.
- **Workers pushed from scratch `worktree-claude-*` branch names 3×** (#2864, #2870, #2873) — filed #2866; the /implement flow needs a hard branch-rename step.
- **`mcx tracked` prNumber/branch didn't auto-populate** — every PR had to be attached manually with `work_items_update` before the phase scripts saw it (16 manual attach calls). The impl phase also kept returning `in-flight` after PR push until triage was run manually. Pre-existing gap, surfaced hard at this scale.
- **Local `bun test` on clean main failed** on stale scratch specs under gitignored `build/` (test-partition glob) — filed #2850, scratch removed.

## Patterns established

- **Destructive-path QA briefs**: any PR that removes worktrees/branches/files gets an enumerated adversarial checklist in the QA spawn note (uncommitted variants, unpushed commits, closed-vs-merged, dry-run parity, fail-closed on comparison error).
- **Race-against-signal is the sanctioned drain-bound shape**: `Promise.race([drain, stateMachineSignal])` is allowed where `Promise.race([drain, setTimeout])` is banned — early-stop only on proven completion. Worth folding into the ws-server module docs (#2881 tracks doc drift).
- **Override comments are living records**: when post-merge evidence contradicted my qa:fail→qa:pass mechanism on #2863, the correction went onto the merged PR — audit trail stays honest even after landing.
