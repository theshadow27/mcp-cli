## The retro headline: we don't dogfood our own transport

User-prompted investigation mid-sprint: **every sprint session runs the legacy
sdk-url/WS path** because the grid pins archived claude-2.1.119 (≤2.1.122
gates to ws). The stdio transport (#2234) is shipped, unit-green, and has
never executed under real sprint load. Its spike's own "#1 MVP risk"
(pipe-buffer deadlock under multi-session concurrency) is untested precisely
in the regime we run nightly. Not dogfooding our own features is an
antipattern — we built a transport and kept driving the old one.

A mid-sprint canary was attempted and aborted: per-spawn `claudeBinary` config
flips are unreliable (race or daemon caching — #2681), and the fallback
(tail-of-sprint canary) would have tested stdio at minimum concurrency,
dodging exactly the regime that matters. Deferred deliberately (user call) to
an early-next-sprint full-concurrency canary — #2688, arguably P1 because
stdio also silently drops the ws containment round-trip (worktree containment,
input rewrite, delegate mode). Next-sprint plan must include it.

## What worked well

- **The investigations gate earned its cost in an unexpected dimension.** The
  issue body already documented the mechanism, so the expected value was low —
  but the nerd-snipe's real contributions were fix-option adjudication and
  blast-radius mapping: it proved #2587/#2588 had NO stale log entries, so the
  plan's blocker edge was precautionary-only. Empirically verified, agent-grid
  lane launched ~1h earlier than the dependency graph allowed.
- **Adversarial review caught what every earlier stage missed**: the UTC
  parsing bug the investigation got wrong (#2675), the NaN bypass disabling
  the very guards under review (#2678), a settle-timer violating the repo's
  core testing rule (#2695), resume-session testing a deleted session (#2691).
  4 of 6 reviews produced blockers; all were real.
- **Micro-repair + fresh-eyes verification as the only repair path.** 5 PRs
  went review → reviewer-self-repair → independent verification. Zero opus
  repair sessions all sprint. Every repair commit was verified by a session
  that didn't write it — including two QA-authored commits that got dedicated
  independent verifiers.
- **Event-driven pipeline at full speed**: first PR merged 12 minutes after
  spawn; monitor-stream notifications meant zero polling; ~30 sessions
  orchestrated with no idle gaps between phases.
- **Contemporaneous observations file** (new pattern): committed to the sprint
  branch during the run. This retro was written from evidence, not memory.
- **Fable worker canary passed**: claude-fable-5 through pinned 2.1.119,
  spawn→PR→merge clean, cost comparable to opus fillers.
- **Backgrounded issue-author agents**: 11 issues filed during the run with
  zero orchestrator context spent on dedup-searching; two correctly enhanced
  existing issues (#2656, #2648) instead of duplicating.

## What didn't work

- **Worker wrote to the orchestrator's main checkout** (#2693). #2463's impl
  session left its pre-repair implementation in the main tree (mtime 03:22:34);
  discovered an hour later when `git pull` aborted. The sentinel blocked
  commits (worked as designed) but nothing guards writes; ws containment
  apparently checks Bash cwd, not Edit/Write path args. Likely trigger:
  absolute main-checkout paths quoted in the investigation comment.
  Mitigation now: `git status --porcelain` on main is part of my mental
  pre-merge hygiene; real fix tracked in #2693.
- **QA sessions pushed code to branches they were judging — twice** (#2463's
  QA: stale-base duplicate, rebased away; #2616's QA: new crash test, needed a
  dedicated independent verifier). qa.md has no "report, don't push" rule —
  the #2651 hardening added trust boundaries but missed this. Filed #2698
  (meta, next plan).
- **Stale-base churn, 3 incidents** (#2679): worktrees fork from local main,
  which lags origin during a fast merge train — caused #2617's resurrected-
  files scare, #2675's duplicate fix commit, and two extra rebase+CI rounds.
- **Committed .mcx.lock was stale since #2570** and every sprint's pre-flight
  `mcx phase install` silently papered over it locally (#2656 evidence +
  #2673 for the worktree-cwd resolution split that exposed it). My own CWD
  drift (a `cd` in a compound command persisting) was the accidental
  discovery mechanism — subshell discipline from then on.
- **Host oversubscription SIGTERM storms** (#2690): ~10 concurrent sessions ×
  parallel am-i-done test phases → 8.9s mass-SIGTERM runs. One worker tried to
  absorb it as "the known CI issue" (wrong — that one is Linux-only); the
  clean-main comparison protocol resolved it without a flake label entering
  any context. The systemic fix (stagger/flock the test phase) is filed.
- **One label-hygiene slip by the orchestrator**: merged #2691 with the stale
  round-1 review:changes attached; audit comment added post-merge instead of
  the pre-merge swap done for #2675/#2695. The pre-merge swap should be
  mechanized in done-fn rather than relying on orchestrator memory.
- **Plan-vs-triage scrutiny disagreed 4×** (churn-based triage said low for
  #2463/#2651, high for #2505). Overrode to the stricter of the two each
  time — right call every time (both "low" ones produced review blockers).
  The plan's scrutiny column should flow into the work item at track time so
  triage can take max(plan, churn) mechanically (#2665's adjacent shape).

## Patterns established

- **Observations file on the sprint branch, written as events happen** — raw
  material for the retro, survives context summarization. Keep.
- **Stricter-of(plan-scrutiny, triage-scrutiny)** as the routing rule.
- **QA-authored commits get a dedicated independent verifier** before merge
  (until #2698 removes the cause).
- **Stale verdict labels swapped pre-merge with an audit comment naming
  mechanism + verifying session** — self-approval distinction made explicit
  in the trail.
- **Micro-repair default**: reviewer fixes its own contained findings; fresh
  session verifies; opus repair reserved for "needs redesign" replies (none
  occurred).
