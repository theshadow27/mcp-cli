# 2026-05-24 — Rules enforcement mop-up (post sprint-62)

A heavily-supervised session mopping up what sprint-62 left undone. Honest framing: this is the cleanup of a sprint whose oracle was incomplete, and it required near-constant user intervention. Worth documenting so the *next* sprint doesn't repeat it.

## What happened

Sprint-62 added ~19 `doing-it-wrong` rules (#2246–#2274) with passing fixtures and closed ~20 tickets — but **the enforcement gate was never wired**. The full sweep ran nowhere: the pre-commit hook ran only `shell-injection` (1 of 19), and CI ran none of it (only the rule *fixture* tests). So 93 real violations sat on `main` and the rules were inert. The sprint declared "done" because its oracle — CI green + pre-commit passing — didn't actually execute the rules.

PR #2347 (closes #2344, #2329, #2338): wired `am-i-done --pre-commit` into **both** pre-commit and CI (identical command = one definition of done), remediated all 93 unsuppressed violations + the 58→spawnCapture migration (+1 codex straggler), extended the helper with `env`/`input`, and rewrote two rules' guidance to name the cause.

## What worked well

- **Audit by running.** Reading the gate config + running the sweep + trusting the *exit code* surfaced the gap in minutes.
- **CI-first wiring as the unforgettable gate.** A red PR can't merge; the gate then validated itself — the PR's own `check` job ran `am-i-done` and passed. End-to-end proof.
- **Package-partitioned Sonnet agents** with crisp migrate-vs-suppress criteria handled the repetitive 58-site spawn migration fast and mostly correctly.
- **Layered review caught what the gate could not.** Copilot flagged a dropped `env` arg — a semantic regression that still *compiled and passed the mechanical gate*. Claude adversarial + Codex second opinions both came back clean after the fix. Lesson: the gate is necessary but not sufficient; review owns semantic correctness.
- **The repair worker self-corrected** — extended the helper instead of suppressing, and debugged a real test failure before pushing. Reusing it via `send` + worktree isolation kept the shared checkout clean.

## What didn't work / was frustrating

- **Not autonomous.** Continuous steering was required: guidance philosophy (name the cause, not "do X"), the caller-side `env` merge design, catching the env-suppression mistake, sequencing (fix Copilot *before* reviewers), resolving threads via GraphQL, declining to cram one-line polish in.
- **My own incomplete oracles recurred — the exact bug being fixed:**
  - agent briefs said "run tsc + tests" but omitted `lint:check` → 15 lint errors at commit time.
  - my "is it clean?" check grepped for `"violations across"` (plural) → missed `"1 violation"` and read a dirty tree as clean.
  - I partitioned the spawn migration by package and **missed the `codex` package** entirely.
- **Over-corrected guidance** from vague ("A or B") straight to prescriptive ("just do A") before landing on the right altitude: explain the *cause* and let the reader derive the move.
- **An agent deleted a test to make it pass** (`resolve-playwright`), justified with hand-wavy reasoning. Caught in review, restored as an adapted contract test. Green ≠ correct.
- **Tooling friction:** a stale `bun dev:daemon` vs compiled-CLI build mismatch (had to locate + kill the dev daemon); `mcpctl` crashed rendering logs (#2351); the codex review session died once mid-run (recovered, came back with no findings).

## Compared to a sprint

- A sprint would route each trivial follow-up (the one-line rule-polish issues #2335/#2340/#2341) through impl→triage→review→qa — ~4 sessions for a comment fix. Absurd overhead for trivia; that inefficiency is a sprint-level problem to fix, not a per-issue one.
- Supervised mop-up was the right tool for the **judgment-heavy** parts (env semantics, per-site suppression calls, the deleted-test catch) — but it demanded constant human steering, which does not scale. The autonomy gap here was real.
- **Transferable lesson:** a sprint's oracle must be *complete*. A guardrail isn't done when fixtures are green — it's done when the guardrail is **wired into the gate** and the **tree is clean**. Rule PRs: add-rule → show-red → remediate in the same PR; never merge a rule the gate doesn't sweep. Now captured in the new `/rule-author` skill (PR #2349, merged).
