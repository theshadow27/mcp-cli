## What worked well

1. **Dependency-ordered parallelization** ‚Äî decomposing #133 into 6 serial layers with layers 0 and 1 parallelized across forked sessions made the large feature manageable. Clear dep graph up front prevented blocked sessions.
2. **`/simplify` after initial commit, before PR** ‚Äî committing first then running simplify gave a clean diff view. Found a real functional bug (`waitForResult` timeout using `indexOf` on object reference instead of comparing wrapped closures) and eliminated 101 lines.
3. **Tools defined in a shared file** ‚Äî extracting `CLAUDE_TOOLS` to `claude-session/tools.ts` after the simplify pass eliminated diverging duplicate tool definitions in both `claude-server.ts` and the worker. The simplify review caught this immediately.
4. **DI with constructor/function params for adapters** ‚Äî all new code (tty adapters, config.ts IPC functions) used injection rather than direct `Bun.spawn`/IPC calls, making coverage achievable without integration overhead.
5. **`terminateSession()` unified cleanup** ‚Äî simplify review caught a race condition where `proc.exited` and WS close could both fire; unified into a single `terminateSession()` that clears keep-alive timer, drains waiters, removes from map.

## What didn't work

1. **QA closed issues without checking CI** ‚Äî PR #178 (SQLite session table) was green on local but CI had been failing for multiple runs. QA agent approved the merge anyway. Had to go back and fix after the fact. Root cause: QA step didn't include `gh pr checks`. Filed + fixed in qa.md.
2. **`fs.watch` on Linux for rename events** ‚Äî `rename(foo.tmp, bar.json)` doesn't reliably emit `bar.json` as the filename on Linux inotify. The production watcher had a latent bug; only surfaced via CI. The fix (matching on basename prefix + null) works but is a workaround for OS behavior.
3. **`extractJsonFlag` doesn't recognize `--json`** ‚Äî it only matches `-j` and `--format json`. Caused a test failure when `--json` was used as a flag in `mcx claude ls`. Caught by test, fixed to use `-j`.
4. **Coverage ratchet requires refactoring for testability** ‚Äî both `config.ts` and the tty adapters required DI refactors to bring coverage above threshold. The ratchet is doing its job, but it added 30-60 min per feature.
5. **`any`-typed mock inference in tests** ‚Äî `callTool` mock typed with 1-arg signature; `toHaveBeenCalledWith` assertions for 2-arg calls needed explicit mock typing to pass typecheck.

## Patterns established

- **`terminateSession()` unified teardown**: consolidate all cleanup (timer clear, waiter drain, map removal, proc kill) into one method called from both `proc.exited` and WS close handlers to avoid race conditions.
- **Tool schema in shared file**: when the same tool definitions need to appear in both a Worker (server-side) and the main thread manager (tool cache), extract to a shared `tools.ts` to prevent divergence.
- **CI check is part of QA**: `gh pr checks --watch` before any merge/close. Added to qa.md as a mandatory step.
- **`startTestDaemon({ dir })` with stale socket cleanup**: test harness for daemon lifecycle tests should handle cleanup of any stale socket at the given dir before spawning ‚Äî extracted as a reusable param.
- **Watcher filename filter with prefix match**: for directory watchers that need to detect atomic writes, match on `filename.startsWith(basename)` (not exact match) and also handle `null` filename ‚Äî the existing hash check prevents false reloads.
