## What worked well

- **QA as the production gate caught what two adversarial review rounds missed.** PR #2706: both review rounds (opus) validated unit specs; QA's dist-binary check found `--claude-binary`/`--transport` were dead code on the real CLI path — tests passed by injecting deps into a path production never takes. The repair added a real-dispatch test. Lesson encoded in QA briefs: "verify through the shipped binary."
- **Self-repair + fresh-eyes verification loop ran 6 times with zero self-approvals.** Reviewers fixed their own contained findings; a session that didn't write the commit verified every one (round-2 reviews re-ran round-1's exact exploits; one re-ran new tests against pre-repair source to prove they fail).
- **Canary protocol held under failure.** Both stdio canaries died identically at spawn (`spawn exited` in daemon log, ws siblings fine) → abort per protocol step 6, no third attempt, evidence on #2688, sprint finished on ws. No global state to roll back because #2681's per-spawn flags were the gate.
- **Fable expansion: 2/2 merged end-to-end** (#2685, #2679), including a genuinely useful host-instability investigation (wedge discovery + #2597 data points + #2666 confirmation). Fable also held the line on gate integrity when offered an easy `--no-verify`.
- **Zero gate bypasses under sustained host duress.** 4 of 5 batch-3 pre-push gates hit Bun worker-crash cascades (load avg ~8.7 with 2-3 wedged day-old test processes); all resolved by load-eased retries; CI on clean runners as arbiter, noted in PR bodies.
- **Issue-author delegation**: 12 new issues + reopen of #1328 + data points on 6 existing issues, all dedup-checked off-context.

## What didn't work

- **stdio canary failed hard** — both workers exited within seconds of spawn, before any protocol traffic. Transport-attributable (3 ws siblings spawned same minute all fine). Worker stderr wasn't captured in the daemon ring buffer, so the *why* is unknown — that capture gap is the actionable part, noted on #2688. Next sprint needs a dedicated investigation before any retry.
- **`.mcx.lock` went stale 4 times mid-sprint**, each time blocking phase runs until a manual `mcx phase install` + sprint-branch commit. Root cause was the sprint's own #2656 finding (entry-only drift hash) interacting with the daemon's post-merge auto-pulls. Fixed structurally by #2728 + the new-format lock; residue is #2737 (worktree-aware check).
- **Orchestrator bypassed phase bookkeeping post-restart.** After the daemon restart I direct-spawned reviewers without ticking `mcx phase run review` first; the new fail-closed freshness checks (correctly) refused to honor their verdicts — `review_spawned_at missing — fail closed`. Worked around via the documented `phase=qa` work-item write. Rule for next sprint: always tick the phase to emit the spawn plan, even when overriding the command; the ACL (#2713) makes the sentinel keys unforgeable by design.
- **Wedged host processes degraded the whole sprint** and the orchestrator's targeted kill was denied by the auto-mode classifier (policy: no killing processes the agent didn't create). Right call per the standing rule, but it left 2-3 cores pinned for the entire sprint; operator cleanup is still pending. The wedges have CPU samples attached to #2597 (spinning at `bun+0x328b324`, `kevent64` — worker blocked on a closed parent IPC channel).
- **mcx logs flag confusion** (`--tail` doesn't exist; it's `--lines` + `--daemon`) and `====` as a zsh echo separator aborting compound commands — minor orchestrator papercuts, no issues filed (first is documented usage, second is shell foot-gun).

## Patterns established

- **QA briefs must include one production-path verification** (dist binary, real CLI dispatch, real provider fixture shapes) — unit-spec green is not evidence the surface exists.
- **Adapter contract tests**: when N adapters target a shared guard/contract, test each adapter's *real* native shape through the shared component; fabricated canonical keys in fixtures mask wiring bugs (suggested as a doing-it-wrong rule in #2720's review).
- **Reviewer rebases DIRTY PRs mechanically** (regenerate, don't hand-merge; flag any judgment call instead of self-approving it) — used twice (#2718, #2728), cleanly both times.
- **Gate-bypass refusals are now memory** (`feedback_never_bypass_gate.md`): retry on tracked-signature flake, stop on new signature, CI as arbiter.
