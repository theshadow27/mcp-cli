## Patterns established

- **`qa:pass` / `qa:fail` PR labels** are the control-plane signal between QA and orchestrator. QA sessions apply exactly one; orchestrator reads via `gh pr view <n> --json labels`. Replaces the ad-hoc "look for 'QA complete' in session output" pattern.
- **Only the orchestrator merges and moves branches.** Orchestrator stays on `main`; `git pull` after each merge keeps local current. This eliminates the "rebase after big changes" class of problem — new spawns always branch from up-to-date main.
- **Meta files are retro-only.** `.claude/skills/**`, `.claude/memory/**`, `CLAUDE.md`, `.gitignore` — never modified by workers during a sprint. Follow-up work uses the `meta` label; `/sprint plan` Step 1a surfaces pending `meta` issues for user review before the next sprint begins.
- **`mcx claude wait --timeout 300000` is the default.** 5 minutes, event-driven, no throughput cost. Drop to 30000 only when actively supervising a specific session.
- **Don't `bye` a QA session until the PR has a `qa:` label.** Same handoff-artifact rule as "don't bye before PR is pushed."
- **`--track <issue>:done` enhancement filed (#1284)** — would let the orchestrator merge + close + untrack + end-session atomically, with clear error messages for every pre-merge gate (CI red, label mismatch, merge conflict).
