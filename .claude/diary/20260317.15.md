## What worked well

- **Batch 1 velocity was excellent**: 5 of 6 Batch 1 issues completed within 30 minutes of spawning. Small, well-scoped issues with clear specifications ship fast.
- **Adversarial review caught real bugs**: Both review rounds for #696 found genuine data corruption issues (file leaks, permanent alias overwrites). The 2-review protocol for high-scrutiny rewrites is validated.
- **Worktree isolation**: After fixing the run.md gap, all subsequent QA sessions ran in proper isolation. No more main folder pollution.

## What didn't work

- **~$2,700 burned on pre-commit retry loops**: Sessions `1667f681` (#779, $108), `9c02b51c` (#782, $469), `efadc9c1` (#787, $1,154), and `745a5378` (#696 repair, $961) got stuck overnight retrying pre-commit hooks. The `watcher.spec.ts` timing budget violation (8.5s vs 5s budget) caused every commit attempt to fail, and sessions retried indefinitely. This is the single biggest cost incident in project history.
- **Test time budget was actively harmful**: The hard-fail budget in `check-coverage.ts` blocked commits for ALL changes when ANY test exceeded 5s, regardless of whether the slow test was related. Timing varies with system load (especially with 5+ concurrent sessions). The exclusion list grew from 3 to 4 entries during the sprint, and more will follow. **Disabled the hard-fail and filed #812 for hash-based replacement.**
- **QA sessions spawned without `--worktree`**: The run.md assumed worktrees would survive from implement through QA, but `bye` auto-cleans worktrees when the branch is pushed. QA sessions defaulted to the main repo folder, checking out branches and modifying files in the user's working tree. Fixed run.md to document `--worktree` fallback.
- **#696 repair session at $961**: The repair itself was straightforward (~$0.80 for the actual work) but the session spent $960 retrying the pre-commit hook. This is the same root cause as above.

## Patterns established

- **Always use `--worktree` for QA/review when implementation worktree was auto-cleaned** — updated run.md with explicit fallback instructions and a CRITICAL warning about never spawning without isolation.
- **Test budget is now warn-only** — `check-coverage.ts` reports slow tests but does not block commits. #812 tracks the hash-based timing cache replacement.
- **Pre-commit hook respects change scope** — #789 added tiered checks so docs-only and config-only changes skip the test suite entirely.
