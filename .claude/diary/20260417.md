## What worked well

1. **Backup before build, note version.** User prompted this before the demo. `cp` the v1.5.0-rc.1 binaries aside, build fresh, document the rollback one-liner. Trivial prep that would have saved the demo if v1.5.1 had regressed.
2. **Demo choreography repros in sprint plan.** The three canned repros in sprint-36.md Act 1 all matched live behavior exactly on the demo build — proof the plan accurately described broken state before the sprint fixed it.
3. **Phase pipeline carried real customer bugs.** Every merged PR went through `impl → triage → [review|qa] → [repair] → done` via `mcx phase run`. No legacy spawns, no manual gating — the pipeline was the demo.
4. **Triage routing was accurate.** Low-scrutiny (#1410, #1411, #1412, #1414) → QA; high-scrutiny (#1413, 3 packages) → review. Both correct: low ones were genuinely clean, and #1413's review caught real bugs.
5. **Micro-repair pattern.** #1413 reviewer self-repaired its own 3 blocking findings in the same session ($1.33) vs spawning a fresh opus repair. Sticky updated to Approve with delta table. Saved a spawn cycle.
6. **`done` phase blocked on CI.** #1427's repair commit introduced a lint failure; `done` returned `ci_not_green` and refused to merge. Safety net worked — caught a gap in the pipeline (QA ran locally before the repair commit).
7. **Parallel sessions under quota.** 3 impls + QAs in flight simultaneously stayed under the 80% 5-hour quota. Peak 6 active sessions.
8. **Auto-merge for trailing PRs.** Sprint stopped after Batch 1 with #1412 still waiting on CI re-run. Armed `gh pr merge --squash --delete-branch --auto` so it lands unattended.

## What didn't work

1. **QA passed before Copilot review arrived.** #1427 got `qa:pass` labeled before Copilot's 2 valid inline comments posted. Orchestrator's 4-surface audit caught it, but it cost a $0.64 repair + $0.10 lint + $0.10 rebase cycle that wouldn't have been needed if QA had waited. Candidate for a "wait for Copilot" step in QA phase.
2. **Repair phase doesn't clear `qa:fail` label.** #1412's repair pushed fixes but left `qa:fail` on. QA phase saw the label and routed back to repair instead of re-running QA. Had to manually swap the label. Repair phase should clear `qa:fail` on entry (or phase script should check commit vs label timestamps).
3. **`done` doesn't auto-rebase.** Three PRs needed rebase after earlier PRs merged to main (#1410, #1413, #1412). Each required a one-shot sonnet spawn (~$0.10–0.13). `done.ts` reports "conflicts" but orchestrator spawns the rebase manually. Long-term fix is merge-queue (#1397).
4. **Session `6586` stuck in `connecting` forever** — no logs, no result. Immediate `bye` per run.md. No root cause; matches #1426 pattern (daemon race condition).
5. **Impl phase defaults to opus regardless of plan.** Phase handler emitted opus for #1412/#1413 even though plan.md specified sonnet. Override at spawn time worked, but the phase script doesn't read the sprint plan's model column. Minor cost inflation.
6. **#1411 went into plan mode, needed manual approval nudge.** Opus turn completed at $0.48 with only 10 turns — had stopped to present a plan. No automatic detection that a session is blocked on plan approval vs actually idle. Required orchestrator intuition.
7. **Flaky `server-pool.spec.ts` (8000ms SIGTERM)** blocked #1412 CI for the full sprint; filed #1433. Had to swap `qa:fail` → `qa:pass` manually and re-run.
8. **Worktree pollution from rebase spawn** — the rebase session checked out a branch in the main worktree. Had to clean up before continuing.

## Patterns established

- **Version-pin + backup before any on-stage build.** `cp ~/.local/bin/{mcx,mcpd,mcpctl} ~/.local/bin/backup-<version>/` then note the rollback command. Standard demo hygiene.
- **Demo mode: run real, but hold late batches.** Batch 1 (5 issues, ~60 min) is a good demo size. Stop after Batch 1 and show release + retro as the "finish" rather than waiting on all batches. Arm auto-merge on trailing PRs.
- **Orchestrator narrates the phase transitions.** `mcx phase run impl → JSON → mcx claude spawn → mcx claude wait → mcx phase run triage → ...` is good screen content because each step is one line with structured output.
- **4-surface Copilot audit before merge is mandatory.** PR body, inline comments, review containers, linked issue. QA can miss inline comments that arrived after `qa:pass` labeled.
- **Self-repair in review session, not a fresh spawn.** If the reviewer finds contained issues, send repair instructions back to the same session and have it update the sticky with a delta. Saves a full opus spawn.
- **When a phase says "conflicts", spawn a one-shot rebase.** Cheap sonnet turn (~$0.10), force-push, wait for CI. Don't try to auto-rebase in the orchestrator.
